Sembunyikan balasan tiket

Saat melihat tiket dengan banyak balasan, Hesk dapat menyembunyikan balasan lama dan hanya menampilkan yang terbaru.

Ini memungkinkan anggota staf melihat balasan terbaru untuk tiket tanpa harus menggulir melewati banyak balasan lama.

Staf dapat melihat balasan yang disembunyikan kapan saja dengan mengklik tautan "Tampilkan balasan sebelumnya".

Panjang baris adalah faktor penting untuk kenyamanan membaca, terutama jika Anda mengharapkan membaca dalam waktu lama.

Buka kembali tiket

Atur ke ON untuk memungkinkan pelanggan membuka kembali tiket setelah tiket ditutup. Ketika diatur ke OFF, setelah ditutup pelanggan harus membuka tiket baru.

Penilaian balasan

Jika diatur ke ON, pelanggan akan dapat menilai balasan staf sebagai Helpful atau Not helpful. Masukan pelanggan ini akan digunakan untuk menilai staf.

Prioritas pelanggan

Jika diatur ke ON, pelanggan akan dapat mengatur prioritas/urgensi untuk tiket mereka mulai dari Rendah hingga Tinggi. Jika diatur ke OFF, semua tiket akan dikirim dengan prioritas default (Rendah) dan hanya staf yang dapat mengubah tingkat prioritas menjadi yang lebih tinggi.

ReCaptcha

ReCaptcha adalah layanan anti-bot gratis dari Google.

Untuk menggunakan ReCaptcha Anda perlu:

  1. Daftar untuk ReCaptcha
  2. Klik Dapatkan reCAPTCHA
  3. Masukkan detail situs web Anda dalam formulir Daftarkan situs baru
  4. Salin Kunci Situs ke pengaturan HESK
  5. Salin Kunci Rahasia ke pengaturan HESK
  6. Simpan pengaturan HESK
  7. Coba kirimkan tiket baru melalui antarmuka pelanggan untuk memverifikasi reCAPTCHA berfungsi dengan baik.

Gunakan pertanyaan anti-SPAM

Alihkan penggunaan pertanyaan keamanan anti-SPAM. Ini adalah cara yang sangat sederhana namun cukup efektif untuk melawan spammer (bukan jika mereka memutuskan untuk menargetkan situs web Anda secara khusus tetapi harus memblokir semua spambot yang menjelajahi web). Ide dasarnya adalah membuat pertanyaan yang unik yang hanya bisa dijawab oleh manusia. Anda dapat menggunakan kode HTML dalam pertanyaan. Jawaban tidak sensitif terhadap huruf besar/kecil. Beberapa contoh:

-> Pertanyaan -> Jawaban
Apa warna air? Biru
Berapa angka berikutnya setelah empat? (gunakan hanya digit) 5
Ketik kode akses ABCDE di sini: ABCDE
Kode akses: (temukan di bagian bawah halaman tautan kami) Beberapa kode

Peringatan! JANGAN gunakan contoh-contoh ini, gunakan pertanyaan unik yang hanya akan digunakan di situs web Anda!

Batas upaya login & Waktu larangan (menit)

Fitur ini membatasi serangan brute-force untuk login ke HESK atau melihat tiket.

Setelah Batas upaya login jumlah upaya gagal untuk melihat tiket atau login ke HESK, alamat IP akan dilarang selama Waktu larangan (menit).

Untuk menonaktifkan pembatasan upaya login yang gagal, atur nilai ke 0 (TIDAK disarankan!).

Bingkai

Aktifkan untuk mencegah HESK dimuat dalam bingkai di situs web pihak ketiga. Ini membantu memerangi upaya "clickjacking".

Jika diaktifkan, HESK akan mengirimkan header HTTP X-Frame-Options: SAMEORIGIN ke browser.

Jika Anda ingin menampilkan HESK di dalam bingkai di situs web dengan nama domain yang berbeda dari yang dipasang HESK, nonaktifkan fitur ini.

Dalam kasus di atas, Anda mungkin juga perlu mengatur pengaturan Cookies ke None dan menggunakan koneksi https://!

 

Tutup jendela

 

 

 

 

 

 

Cookies

Pengaturan ini mengontrol atribut SameSite dari cookie HTTP. Untuk penjelasan terperinci, lihat artikel ini.

Singkatnya:

Strict - cookie hanya akan dikirim ke permintaan di bawah nama domain Anda. Mengikuti tautan email, misalnya, mungkin memaksa Anda untuk masuk kembali.

Lax - nilai default Hesk. Cookie tidak dikirim pada subrequest lintas situs normal (misalnya, untuk memuat gambar atau bingkai ke situs pihak ketiga) tetapi dikirim saat pengguna menavigasi ke situs asal (mis., saat mengikuti tautan).

None - cookie akan dikirim ke permintaan apa pun, tetapi hanya di bawah koneksi aman (https://). Gunakan pengaturan ini jika Anda menampilkan Hesk dalam bingkai di domain pihak ketiga.

Jika Anda ingin menampilkan HESK di dalam bingkai di situs web dengan nama domain yang berbeda dari yang dipasang HESK, atur ke None dan gunakan koneksi https://.

SSL

Anda dapat memaksa HESK untuk menggunakan koneksi aman (https versus http).

Untuk mengelola pengaturan ini, pertama-tama buka file admin_settings_help_desk.php di browser Anda menggunakan https://, misalnya:

https://example.com/hesk/admin/admin_settings_help_desk.php

Ini untuk memastikan bahwa SSL dikonfigurasi dengan benar dan berfungsi di server sebelum mengatur HESK untuk memaksa menggunakannya.

Kunci Akses URL

Gunakan Kunci Akses URL untuk mencegah akses web tidak sah ke file cron Hesk Anda, seperti:
- /inc/mail/hesk_imap.php,
- /inc/mail/hesk_pop3.php,
- /cron/email_overdue_tickets.php, dll.

Ketika kunci akses diatur dan seseorang ingin mengakses salah satu file tersebut melalui alamat URL (browser web), mereka perlu menentukan kunci yang benar dengan menambahkan ?key=XXXXXXXX ke alamat URL. Contoh:

https://example.com/hesk/inc/mail/hesk_imap.php?key=XXXXXXXX

Kunci akses Anda harus berupa string acak dengan panjang minimal 20 karakter. Karakter kunci akses yang valid adalah: a-z A-Z 0-9 _ . -

Kunci akses tidak digunakan saat file cron diakses melalui antarmuka baris perintah, hanya saat disajikan melalui http(s).

Jika Anda membiarkan Kunci Akses URL kosong, itu tidak akan diperlukan/digunakan.

Otentikasi Multi-Faktor (MFA)

Gunakan otentikasi multi-faktor untuk mengharuskan pengguna mengotentikasi dengan kata sandi mereka bersama dengan metode otentikasi sekunder.

- Mengaktifkan "Otentikasi Multi-Faktor yang Diperlukan" akan mengaktifkan MFA untuk pengguna baru dan yang sudah ada.

- Menonaktifkan "Otentikasi Multi-Faktor yang Diperlukan" tidak akan menonaktifkan MFA untuk pengguna yang sudah ada; itu hanya akan mempengaruhi pengguna baru.

Ekstensi file yang diizinkan untuk melampirkan file. Untuk menambahkan lebih banyak ekstensi gunakan format yang sama .ext dan pisahkan dengan koma. Pengaturan ini hanya tersedia jika "Gunakan lampiran" diatur ke YA.

Akun pelanggan

Mendaftarkan akun memungkinkan pelanggan mengelola tiket mereka di satu lokasi dan memberi Anda kendali atas siapa yang dapat mengirim tiket dukungan.

Opsi dijelaskan:

  • Nonaktif
    Pelanggan tidak harus mendaftar atau masuk untuk mengirim, melihat, dan menanggapi tiket dukungan.

    Perhatikan bahwa tiket dukungan tetap pribadi; pelanggan mengaksesnya menggunakan ID pelacakan unik dan (opsional) alamat email mereka.
     
  • Diaktifkan - Opsional
    Membuat akun pelanggan adalah mungkin dan opsional untuk kenyamanan mereka. Pelanggan dapat mengirim tiket dukungan meskipun mereka tidak memiliki akun.
     
  • Diaktifkan - Diperlukan untuk tiket
    Hanya pelanggan dengan akun yang valid yang akan dapat mengirim tiket dukungan. Fitur meja bantuan lainnya (basis pengetahuan, pesan layanan) bersifat publik.
     
  • Diaktifkan - Diperlukan untuk mengakses
    Hanya pelanggan yang masuk dengan akun yang valid yang akan dapat mengakses fitur meja bantuan mana pun.

    Perhatikan bahwa pesan layanan yang secara khusus diatur untuk ditampilkan pada halaman login dan pendaftaran pelanggan (jika pelanggan dapat mendaftar akun) akan bersifat publik.

 

Tutup jendela

 

 

 

 

Pendaftaran akun

Pengaturan ini memberi Anda kendali atas bagaimana akun pelanggan dibuat.

Opsi dijelaskan:

  • Pelanggan dapat mendaftar akun
    Pelanggan dapat mendaftar akun dan memverifikasinya melalui email.
     
  • Pelanggan dapat mendaftar akun, staf harus menyetujui pendaftaran baru
    Pelanggan dapat mendaftar akun dan memverifikasinya melalui email. Anggota staf harus meninjau dan menyetujui semua akun sebelum pelanggan dapat masuk.
     
  • Hanya staf yang dapat mendaftarkan akun pelanggan
    Hanya staf yang dapat mendaftarkan akun pelanggan; pelanggan tidak dapat mendaftar akun sendiri.

Kode batang

Jika diaktifkan, Hesk akan menampilkan kode batang saat mencetak tiket.

Kode batang dapat disembunyikan dari pelanggan (dicetak hanya dari sisi staf).

Parameter:

Tipe - pilih jenis kode batang yang akan dicetak
Format - pilih format tampilan/gambar di halaman HTML
Lebar - lebar gambar kode batang dalam piksel
Tinggi - tinggi gambar kode batang dalam piksel
Warna - warna data kode batang
Latar belakang - warna latar belakang kode batang

Warna dalam notasi Web (nama warna, atau kode heksadesimal, atau sintaks CSS): black, white, red, #000000, #ffffff, #ff0000...
